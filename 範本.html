核心理念：為什麼我們需要 Agent Skills？

痛點突破：

過去 AI 依賴員工個人的 Prompt 能力，品質參差不齊；若將規則寫死在系統程式碼裡，每次變動都要重新上線，曠日廢時。

我們要把關鍵規則從「個人技巧」升級為「可治理的制度資產」，確保任何人、任何系統都得到一致結果。

業務與程式「解耦」

  • 讓規則歸業務單位管理：修改提示/知識庫
  • 讓系統歸 IT 管：維護 API 連線
  • 實現真正的敏捷治理，規則異動零時差

建構模組化知識庫

  • 把資深同仁的「經驗」轉化為知識庫
  • AI 視情境「看目錄挑書」，動態載入規則

第一類場景：文件與對上溝通 (產出品質穩定)

核心價值：長官關心點高度一致，退件原因高度重複。把退件經驗變成系統化制度。

🌟 案例一：請購簽呈檢查與補強

常見退件原因：沒說清楚必要性、沒比較替代方案、缺乏量化效益 (ROI)。

Skill 作用：「長官視角」不是臨時提示詞，而是已固化成 Skill 的檢查規則；AI 會依文件類型自動套用對應規則，逐項點出缺漏並要求補齊（ROI、替代方案、風險與控管等）。

案例二：高層專用摘要模板

痛點：一般 AI 摘要常漏掉高層真正要做決策的關鍵資訊。

Skill 作用：偵測到「高層摘要」需求即 自動載入摘要 Skill （固定欄位/必填關鍵字/禁寫廢話），產出「可決策的一頁版」，而非一般摘要。

PII + GenAI應用

我們可以用[代碼]來取代原本的個人資料，
將處理後的內容送進 GenAI 生成
（擷取指定欄位、翻譯、摘要…等等，請至「馬上試用」體驗），
最後再把[代碼]還原成原始個資，就能達到無痛遮蔽。
例如，當 AI 需要擷取健檢報告中的被檢查人、地址等資訊，或翻譯對話記錄、摘要重點時，若希望最終結果保留個資，就必須將個資還原。
